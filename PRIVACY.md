# Приватность ШТАБа

Коротко: **ничего не уходит наружу.** ШТАБ — локальный инструмент, он работает на твоей
машине под твоим пользователем и не отправляет твои данные никому.

## Что ШТАБ НЕ делает

- **Нет телеметрии.** ШТАБ не считает запуски, не шлёт «аналитику использования», не
  отмечается ни на каком сервере. В коде нет ни одного обращения к чужому серверу.
- **Нет аккаунтов.** Не нужно регистрироваться, входить, привязывать почту. Приложение
  не знает, кто ты.
- **Нет облака.** Твои проекты, статусы, заметки и настройки лежат у тебя на диске
  (папка `data/` и папки твоих проектов). ШТАБ никуда их не копирует.
- **Нет сбора ошибок.** Падения и логи остаются на твоей машине; никакой «отправить отчёт
  разработчику» нет.

## Куда вообще ходит трафик

Только туда, куда ты сам направил:

1. **К твоему ИИ-помощнику.** Терминал и агенты запускают настоящий Claude Code / Codex /
   Gemini CLI — ровно тот, что стоит у тебя. Он общается со своим сервером под твоей
   подпиской, как если бы ты запустил его руками. ШТАБ здесь только посредник-окно.
2. **К твоему git-серверу.** Синхронизация — это обычный `git pull` / `git push` тех
   удалёнок, которые прописаны в твоих же проектах. Никакого «сервера ШТАБа» не существует.
3. **Фоновые обои** (если включены) — картинки с Unsplash. Можно поставить свои локальные
   и трафика не будет вовсе.

Никаких других сетевых адресатов у приложения нет.

## Терминал и агенты — честно

Терминал и фоновые агенты дают ИИ-помощнику доступ к твоим файлам — **ровно такой же**,
как если бы ты открыл терминал и запустил его сам. Это не «дыра ШТАБа», это суть
инструмента. При этом:

- Агент правит **только свою копию** проекта (отдельная ветка git worktree). Твой код он
  не трогает, пока ты не нажмёшь «Принять».
- Никаких `--dangerously-skip-permissions`, никаких молчаливых авто-коммитов.
- Встроенный сайдкар-терминал слушает только `127.0.0.1` и требует токен — сосед по сети
  до него не достучится.

## Веб-пульт (если решишь включить)

В коде есть заготовка «открыть пульт с телефона». По умолчанию она **выключена**. Пока в
ней не закрыты вопросы безопасности сетевого доступа — не включай её в общей/публичной сети.
Это осознанно оставлено на твоё решение.

## Проверить самому

Код открыт (MIT). Сетевые вызовы легко найти поиском по `http`, `fetch`, `net`, `https` —
их нет, кроме перечисленного выше. Не веришь на слово — посмотри.
